# Frontend Technical Documentation & Requirements

## 1. Общая архитектура

- **Фреймворк:** [React](https://react.dev/) (рекомендуется версия 18+)
- **Язык разработки:** [TypeScript](https://www.typescriptlang.org/)
- **Менеджер пакетов:** [pnpm](https://pnpm.io/) / [npm](https://www.npmjs.com/) / [yarn](https://yarnpkg.com/) (на выбор, но рекомендован pnpm)
- **Система сборки/рендеринга:** [Next.js](https://nextjs.org/) (рекомендуется для SSR и удобной маршрутизации)
- **UI-библиотека:** [Material UI (MUI)](https://mui.com/) для всех ключевых компонентов
- **Графики и аналитика:** [Recharts](https://recharts.org/en-US/) или [Chart.js](https://www.chartjs.org/)
- **Иконки:** [Material Icons](https://mui.com/material-ui/material-icons/)
- **Стилизация:** Emotion (встроенный в MUI) или Styled Components

---

## 2. Структура приложения
/src
/components       # Переиспользуемые UI-компоненты
/pages            # Страницы Next.js роутинга (например, /dashboard, /budget)
/modules          # Логика для отдельных блоков (habit, todo, budget, auth)
/hooks            # Кастомные хуки
/utils            # Вспомогательные функции, утилиты
/types            # Определения TS-типов и интерфейсов
/assets           # Логотипы, иконки, картинки
/styles           # Глобальные стили (если требуется)

---

## 3. Основные страницы

- `/login`               — Страница входа по паролю
- `/dashboard`           — Главный дашборд аналитики
- `/budget`              — Управление бюджетом, траты, доходы, категории, кошельки
- `/habits`              — Трекер привычек, календари, прогресс и визуализации
- `/todo`                — Туду-лист, календарь, задачи по дням/неделям/месяцам
- `/settings` *(опционально)* — Настройки профиля, темы, прочие опции

---

## 4. Навигация и роутинг

- Использовать встроенный роутинг Next.js (файловая структура `/pages`)
- Для защищённых страниц реализовать проверку авторизации (Protected Route Pattern)
- Навигационное меню (SideBar или AppBar) доступно после логина

---

## 5. Состояние приложения

- Хранить глобальное состояние с помощью [Zustand](https://zustand-demo.pmnd.rs/) или [Redux Toolkit](https://redux-toolkit.js.org/) (Zustand проще, если проект небольшой)
- Для форм — [React Hook Form](https://react-hook-form.com/) + MUI интеграция

---

## 6. Безопасность и аутентификация

- Для MVP: хранить пароль пользователя зашифрованно в localStorage/sessionStorage/cookies
- Желательно использовать jwt или httpOnly cookies (в будущем)
- Разделяй логику авторизации как отдельный модуль
- Проверять наличие сессии на всех защищённых страницах

---

## 7. Работа с данными

- Для хранения и управления данными использовать облачный сервис [Supabase](https://supabase.com/) (бесплатный тариф).
- Все данные пользователя (бюджет, привычки, задачи и т.д.) должны храниться в отдельных таблицах Supabase, с возможностью расширения структуры.
- Интеграция с фронтендом осуществляется через официальный JavaScript SDK [`@supabase/supabase-js`](https://supabase.com/docs/reference/javascript).
- Операции по созданию, чтению, обновлению и удалению (CRUD) реализовать через вызовы Supabase API.
- Доступ к данным — только для авторизованного пользователя, используя механизмы безопасности Supabase (row-level security, policies).
- Для аутентификации можно использовать встроенные возможности Supabase Auth (email+password), предусмотреть возможность сменить или улучшить механизм аутентификации в будущем.
- Все действия пользователя (добавить, изменить, удалить, отметить выполненным и пр.) должны синхронно обновлять как локальное состояние, так и данные в Supabase.
- Предусмотреть обработку ошибок при взаимодействии с удалённым сервером (отображать сообщения пользователю).
- Для работы в офлайн-режиме (если потребуется) кешировать данные в localStorage и синхронизировать с Supabase при восстановлении соединения.
- Документацию по интеграции Supabase и лучший практики см. [здесь](https://supabase.com/docs/guides/getting-started/quickstarts/react).

**Пример инициализации Supabase:**
```typescript
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = 'https://YOUR_PROJECT.supabase.co'
const supabaseKey = 'YOUR_PUBLIC_ANON_KEY'
export const supabase = createClient(supabaseUrl, supabaseKey)

## 8. UI и стилизация

- Только компоненты MUI, использовать MUI Theme Customization для настройки цветов, отступов, шрифтов
- Шрифт по умолчанию: Inter, fallback — Roboto
- Иконки — Material Icons везде, где возможно
- Графики — стандартные компоненты из Recharts/Chart.js, стилизовать под MUI

---

## 9. Адаптация и отзывчивость

- Обязательно поддержка мобильных устройств (использовать MUI Grid и Breakpoints)
- Минимизировать необходимость горизонтального скроллинга

---

## 10. Код-стайл

- Использовать [ESLint](https://eslint.org/) + [Prettier](https://prettier.io/) (конфиг для React+TypeScript)
- Все компоненты — функциональные, каждый компонент в отдельном файле
- Вынести реиспользуемую логику в хуки

---

## 11. Разработка и расширение

- Организовать код так, чтобы легко было добавить мульти-пользовательский режим без больших переделок
- Заложить структуру под возможную интеграцию с API/BaaS (например, Firebase, Supabase, собственный бэкенд)
- Все бизнес-логика в отдельных слоях (не в компонентах UI)

---

## 12. Другие рекомендации

- Импорт/экспорт данных (CSV, JSON) — предусмотреть простые утилиты
- Фолловить accessibility-гайды (MUI помогает с этим “из коробки”)
- Везде поддерживать loading/success/error-стейт

---

## Рекомендуемый стек

- **Фреймворк/Рендеринг:** Next.js 14+ (React 18+, TypeScript)
- **UI:** Material UI (MUI) + Emotion (или Styled Components)
- **Графики:** Recharts / Chart.js
- **Global State:** Zustand или Redux Toolkit
- **Формы:** React Hook Form (+ yup для валидации)
- **API коммуникация:** fetch или axios (когда потребуется)
