# Roadmap проекта

## Этап 1. MVP (минимально жизнеспособный продукт)

### 1.1. Базовая структура и инфраструктура
- Настроить репозиторий, окружение, линтеры, Prettier.
- Создать базовый проект на Next.js + Typescript + MUI.
- Настроить роутинг на основные страницы: /login, /dashboard, /budget, /habits, /todo.

### 1.2. Авторизация
- Реализовать простую страницу входа по паролю (Supabase Auth: только email+password).
- Ограничить доступ ко всем остальным страницам без авторизации.

### 1.3. Архитектура данных и интеграция с Supabase
- Настроить проект в Supabase (создать проект, получить ключи).
- Создать базовые таблицы: users, budgets, categories, wallets, habits, habit_entries, todos, mood_log.
- Включить Row-Level Security для всех таблиц.

### 1.4. Базовые функции основных разделов

#### Dashboard
- Реализовать отображение: • графика трат, • топ 3 категории, • сумму по кошелькам, • список сегодняшних задач, • напоминание по привычкам.

#### Финансы
- CRUD-операции для категорий и кошельков.
- Ведение истории операций (расходы/доходы): добавление, редактирование.
- Графики и сводная информация.

#### Привычки
- Добавление/редактирование привычек.
- Календарь отслеживания для привычек (отметить сделано/частично/пропуск).
- Календарь настроения.
- Просмотр истории прошлых месяцев в архиве.

#### Задачи
- Добавление, редактирование, удаление задач.
- Календарь с переключением режимов (день/неделя/месяц).
- Возможность задавать период выполнения и приоритет.

## Этап 2. Углубление и доработка

### 2.1. Продвинутые функции
- Возможность комментировать дни в календаре настроений.
- Ведение подзадач (чек-листы).
- Продвинутая фильтрация и поиск по записям (финансы, задачи).

### 2.2. UI/UX улучшения
- Реализовать поддержку тёмной темы, настройка пользовательских тем через MUI.
- Улучшить адаптацию для мобильных устройств.
- Добавить объяснение новых функций/хинты (tooltips, onboarding).

### 2.3. Импорт / экспорт данных
- Реализовать экспорт бюджета/привычек/задач в CSV/JSON.
- Импорт данных из форматов CSV/JSON (минимальная валидация).

### 2.4. Уведомления (basic)
- Push-напоминания о невыполненных привычках или просроченных задачах (если позволяет платформа).
- Email-уведомления о важных событиях/задачах (опционально).

## Этап 3. Бизнес-логика и масштабирование

### 3.1. Продвинутые сценарии и аналитику
- Динамические отчёты и сложные фильтры.
- Сравнения по месяцам, целям, привычкам.

### 3.2. Многопользовательский режим (опционально)
- Перевести весь проект на поддержку мульти-аккаунтов.
- Владелец = админ, добавить разграничение ролей/прав.

### 3.3. Масштабирование и интеграции
- Связка с внешними календарями (Google Calendar).
- Интеграция с мобильным приложением/Push API.
- Возможность подключать сторонние источники данных (например, импорт из банков).

## Этап 4. Поддержка, обратная связь и развитие

- Сбор обратной связи пользователей через отдельную форму.
- Ведение документации по API, миграциям, архитектуре.
- Оптимизация производительности.
- Автоматизация бэкапов и мониторинг состояния базы данных.
