# Включение API для таблиц в Supabase

## Проблема
В Supabase Dashboard на вкладке Policies у таблиц стоит пометка **"API DISABLED"**.

## Что это значит?
Таблица существует, RLS включён, политики созданы, но **таблица не доступна через REST API** (PostgREST). Поэтому все запросы из приложения возвращают 403.

## Решение

### Способ 1: Включить API для каждой таблицы (рекомендуется)

1. Откройте **Database → Tables**
2. Выберите таблицу (например, `wallets`)
3. Нажмите на вкладку **"API"** (рядом с "Data", "Policies", "RLS")
4. Найдите переключатель **"Enable API"** (или "Public API")
5. Включите его (переведите в положение ON)
6. Повторите для всех таблиц:
   - `wallets`
   - `categories`
   - `transactions`
   - `habits`
   - `habit_entries`
   - `mood_log`
   - `todos`

### Способ 2: Включить API через SQL (массово)

Выполните в **SQL Editor**:

```sql
-- Включить API для всех таблиц
ALTER TABLE wallets ENABLE ROW LEVEL SECURITY;
ALTER TABLE categories ENABLE ROW LEVEL SECURITY;
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;
ALTER TABLE habits ENABLE ROW LEVEL SECURITY;
ALTER TABLE habit_entries ENABLE ROW LEVEL SECURITY;
ALTER TABLE mood_log ENABLE ROW LEVEL SECURITY;
ALTER TABLE todos ENABLE ROW LEVEL SECURITY;

-- Также убедитесь, что таблицы включены в API (это делается через UI)
-- В Supabase нет SQL-команды для включения API, только через интерфейс
```

**Примечание**: Включение API через SQL невозможно - это настройка интерфейса PostgREST. Нужно включать вручную для каждой таблицы.

### Способ 3: Проверка глобальных настроек API

1. Перейдите в **API → PostgREST**
2. Убедитесь, что **"Enable API"** включён глобально
3. Проверьте, что нет ограничений на конкретные таблицы

---

## Почему API может быть отключён?

По умолчанию Supabase может отключать API для новых таблиц, если:
- Не включена глобально
- Настройки безопасности
- Ручное отключение

---

## После включения API:

1. **Обновите страницу** в браузере
2. **Выйдите и войдите заново** (чтобы обновить токен)
3. **Попробуйте создать кошелёк**:
   - Откройте Budget → Кошельки
   - Нажмите "Добавить"
   - Заполните и сохраните

**Ожидаемый результат**:
- ✅ Запросы возвращают 200/201
- ✅ Данные сохраняются
- ✅ Появляются уведомления об успехе

---

## Проверка

Откройте DevTools → Network:
- Фильтруйте по "wallets"
- При создании кошелька должен быть POST запрос
- Статус: **201 Created** (не 403)
- В ответе: данные созданного кошелька

---

## Краткая инструкция:

1. **Database → Tables** → выберите таблицу
2. Вкладка **"API"**
3. Переключатель **"Enable API"** → **ON**
4. Повторите для всех 7 таблиц
5. Перезайдите в приложение
6. Протестируйте

---

**Важно**: RLS и API - разные настройки:
- **RLS** (Row Level Security) - контроль доступа к строкам
- **API** - доступность таблицы через REST API

Без включенного API таблица недоступна из приложения, даже если RLS настроен правильно.
